## Agent Rules（来自 `Agent Rules.txt`，2026-07-15 嵌入）

**以第一性原理！从原始需求和问题本质出发，不从惯例或模板出发。**

1. 不要假设我清楚自己想要什么。动机或目标不清晰时，停下来讨论。
2. 目标清晰但路径不是最短的，直接告诉我并建议更好的办法。
3. 遇到问题追根因，不打补丁。每个决策都要能回答"为什么"。
4. 输出说重点，砍掉一切不改变决策的信息。
5. 调用 API 时需要控制 RPM < 200，TPM < 10,000,000。
6. 每次进行开发或调研类任务都需要留下过程开发文档存入 `dev-docs/` 文件夹内，并采用 `序号-内容-YYYY-MM-DD.md` 的格式命名（如 `1-action-checklist-2026-06-25.md`）。
7. 调研过程中需要维护一份 XML / JSON 文件，用于给出参考资料的具体地址与参考的具体相关信息，便于未来其他 agent 查证。
8. 代码开发工作：所有 Agent 书写代码都放在 `src/` 目录下；如当前方法连续 5 个报错，及时退出自动模式并生成一份简要 debug 报告，要求人工手动确认才继续工作。
9. 当前电脑环境：Ubuntu 22.04.5 LTS，`XDG_SESSION_TYPE=x11`；CPU 13th Gen Intel® Core™ i5-13600KF × 20；内存 32 GB；显卡 RTX 4060。
10. 开发以迅捷开发为主，主要进行 demo 开发与功能验证，不允许进行长时间消耗大量 token 的测试，最多只允许进行小规模验证性测试。
11. 每次给出参考方案让用户选择时，需要同时给出各方案的难易程度以及风险点。
12. 每次进行复制/删除等操作前一定要先提交一次 git 避免错误删除无法复原。
